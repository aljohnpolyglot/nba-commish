# Shared UI Components

Reusable presentational components live in `src/components/shared/ui/`.
Import them into any view — do not duplicate the pattern locally.

---

## TabBar
**`src/components/shared/ui/TabBar.tsx`**

Horizontal tab navigation bar. Active tab gets an indigo bottom border.

```tsx
import { TabBar } from '../../shared/ui/TabBar';

<TabBar
  tabs={[
    { id: 'overview', label: 'Overview' },
    { id: 'log',      label: 'Game Log' },
    { id: 'awards',   label: 'Awards' },
  ]}
  active={activeTab}
  onChange={setActiveTab}
  className="px-4 mt-2"   // optional
/>
```

**Used in:** `PlayerBioView`

---

## SortableTh
**`src/components/shared/ui/SortableTh.tsx`**

Sortable `<th>` cell. Renders ↓/↑ next to the label when active.

```tsx
import { SortableTh } from '../../shared/ui/SortableTh';

<SortableTh
  field="pts"
  label="PTS"
  sortField={sortField}
  sortDir={sortDir}
  onSort={handleSort}
  align="right"
/>
```

**Used in:** anywhere with sortable stat tables

---

## GameSimulatorScreen
**`src/components/shared/GameSimulatorScreen.tsx`**

Full-screen live game simulation view (play-by-play, box score, score ticker).
Launched from DayView when the user clicks "Watch Game".

# Architecture Notes

### Non-NBA Player Bios (`nonNBACache.ts`)
Stats and bio hero data for B-League, Euroleague, and PBA players are loaded
lazily from GitHub Gist JSON files. The gist URLs live at the top of `nonNBACache.ts`.
All three leagues now use the **flat stats format** (same fields as RealGM exports:
`ppg`, `rpg`, `apg`, `height`, `weight`, `born`, `draft`, etc.) and are parsed by
`buildFlatEntry()`. Images are **not** read from gists — the player's existing
`imgURL` is preserved.

### Social Posts for Transactions
Signing and trade Charania posts are generated by standalone builders in
`src/services/social/templates/charania.ts`:
- `buildShamsSigningPost(playerName, teamName, teamAbbrev, ovr, prevTeam?, prevLeague?, salary?)`
- `buildShamsTradePost(teamAName, teamAAbbrev, teamBName, teamBAbbrev, assetsToB[], assetsToA[])`

These fire regardless of LLM enabled/disabled. See `playerActions.ts` and
`tradeActions.ts` for the wiring pattern.

### OT Detection
`GameResult.isOT` (boolean) and `GameResult.otCount` (number) are the authoritative
fields. They are used in:
- `DayView` score label ("Final OT" / "Final 2OT")
- `PlayerBioView` game log result column
- `StatisticalFeatsView` result column
